# Gap Analysis

## Strong current areas

- Core tokens and themes.
- Component primitives.
- Overlay primitives.
- Data grid foundation.
- State and adapter architecture.
- Native-first package direction.

## Missing or weak areas

| Gap | Impact | Priority |
| --- | --- | --- |
| App shell/navigation | Hard to build full portals | High |
| Tabs/breadcrumbs/sidebar/top nav | Common admin/customer navigation | High |
| Form completion | Enterprise forms incomplete | High |
| Data display primitives | Detail pages rely on ad hoc layout | Medium/high |
| Toast/progress | Feedback loops incomplete | Medium |
| Overlay robustness | Menus/dialogs may fail in complex layouts | Medium/high |
| Grid server mode | Needed for API-backed data | High |
| Grid saved views | Enterprise preference workflow missing | Medium/high |
| Advanced filters | Needed for dense data apps | Medium/high |
| Virtualization | Needed only after real large data pressure | Later |
| Accessibility QA | Needed before stable release | High |
| Public API docs | Needed for external reuse and AI usage | High |

## Recommended next work

1. Documentation and token reference.
2. App shell/navigation.
3. Form completion.
4. Feedback system.
5. Data display primitives.
6. Overlay robustness.
7. Grid server mode.
8. Export request preview.
9. Saved views.
10. Advanced filters.
