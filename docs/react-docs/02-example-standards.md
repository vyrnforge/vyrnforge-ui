# Example Standards

## Every example should include

1. Problem/use case.
2. Package imports.
3. Live preview.
4. Minimal code snippet.
5. Accessibility notes.
6. Theming notes.
7. When to use / when not to use.

## Example categories

| Type | Purpose |
| --- | --- |
| Component example | Shows a single component and variants. |
| Composition example | Shows multiple primitives together. |
| Pattern example | Shows a realistic enterprise screen. |
| Data example | Shows data-heavy interactions. |
| AI example | Shows exact code AI can copy/adapt. |

## Code snippet rule

Keep snippets short and accurate.

Avoid huge mock datasets inside snippets. Link reusable data files instead.

## AI example rule

AI examples should include:

- component name
- package import
- required CSS imports
- minimal props
- controlled/uncontrolled note
- accessibility requirement
