# Codex Prompt - React Docs App Implementation

```txt
You are working inside the Dravyn UI repository.

Task:
Create or refactor the React docs/playground app so it documents Dravyn UI for humans and AI.

Important:
- Do not add external UI frameworks.
- Do not add routing libraries unless already present.
- Keep native React + TypeScript + CSS.
- Do not implement new library components unless needed for documentation shell.

Goal:
Create a docs/playground experience with sections for Overview, Core, Components, Data Grid, Patterns, and AI.

Required sections:
- Overview
- Core: tokens, themes, density, overrides
- Components: actions, typography, forms, feedback, overlays, layout, navigation, data display
- Data Grid: basic, columns, filters, selection, grouping, server mode, export request, saved views
- Patterns: admin shell, resource detail, settings, form page, data-management page
- AI: context, package map, coding rules, prompt examples

Each page should include:
- title
- purpose
- required imports
- live example
- code snippet
- accessibility notes
- AI usage notes

Acceptance criteria:
- Examples are not all tables.
- Component docs are separate from grid docs.
- AI section exposes how agents should use the library.
- build/playground validation passes.
```
